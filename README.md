这个模板演示了一个使用 [LangGraph.js](https://github.com/langchain-ai/langgraphjs) 实现的简单聊天机器人，展示了如何开始使用 [LangGraph Server](https://langchain-ai.github.io/langgraph/concepts/langgraph_server/#langgraph-server) 以及可视化调试 IDE [LangGraph Studio](https://langchain-ai.github.io/langgraph/concepts/langgraph_studio/)。

核心逻辑定义在 `src/agent/graph.ts` 中，展示了一个简洁的聊天机器人：它在回复用户问题的同时，能够保留并利用历史消息上下文。

## 功能说明

这个简单聊天机器人会：

1. 接收用户 **message** 作为输入
2. 维护会话历史
3. 返回一个占位响应，并更新会话历史

该模板提供了一个可轻松定制和扩展的基础，用于构建更复杂的对话式智能体。

## 快速开始

1. 安装 [LangGraph CLI](https://langchain-ai.github.io/langgraph/concepts/langgraph_cli/)。

```bash
npx @langchain/langgraph-cli
```

2. 创建 `.env` 文件。虽然这个入门应用本身不强制要求密钥，但如果你后续要接入 LLM 提供商或其他集成，通常需要提供 API Key。

```bash
cp .env.example .env
```

3. 如有需要，可在 `.env` 文件中添加你的 LangSmith API Key。

```
LANGSMITH_API_KEY=lsv2...
```

## 如何自定义

1. **添加 LLM 调用**：你可以从 [LangChain.js 生态](https://js.langchain.com/docs/integrations/chat/) 中选择并安装聊天模型封装器，也可以在不依赖 LangChain.js 的情况下直接使用 LangGraph.js。
2. **扩展图结构**：聊天机器人的核心逻辑位于 [graph.ts](./src/agent/graph.ts)。你可以修改该文件来添加新节点、新边，或调整对话流程。

你还可以通过以下方式扩展此模板：

- 添加[自定义工具或函数](https://js.langchain.com/docs/how_to/tool_calling)来增强聊天机器人的能力。
- 实现额外逻辑，用于处理特定类型的用户问题或任务。
- 通过集成[外部 API 或数据库](https://langchain-ai.github.io/langgraphjs/tutorials/rag/langgraph_agentic_rag/)加入检索增强生成（RAG）能力，以提供更定制化的回答。

## 开发调试

在迭代图逻辑时，你可以编辑历史状态，并从之前的状态重新运行应用，以调试特定节点。通过热重载（hot reload），本地修改会自动生效。你可以尝试：

- 修改系统提示词，为聊天机器人赋予独特风格。
- 在图中添加新节点，构建更复杂的对话流程。
- 实现条件逻辑，以处理不同类型的用户输入。

后续请求会追加到同一线程中。你可以点击右上角的 `+` 按钮创建全新线程，从而清空历史上下文。

更多高级功能和示例请参考 [LangGraph.js 文档](https://langchain-ai.github.io/langgraphjs/)。这些资源可以帮助你将模板适配到具体场景，并构建更复杂的对话式智能体。

LangGraph Studio 还可与 [LangSmith](https://smith.langchain.com/) 集成，用于更深入的链路追踪与团队协作，帮助你分析并优化聊天机器人的表现。
